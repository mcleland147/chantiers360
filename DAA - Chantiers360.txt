Document d’architecture applicative – Chantiers360
Version MVP
1. Objectif
Ce document décrit l’architecture cible de l’application Chantiers360 pour le MVP.
L’objectif est de disposer d’une application web interne permettant de gérer les chantiers, les réserves, les photos, les affectations, les alertes et les tableaux de bord.

2. Architecture cible
2.1 Vue d’ensemble
Architecture proposée :
	•	Frontend : React
	•	Backend : Node.js / Express ou NestJS
	•	Base de données : PostgreSQL
	•	Stockage fichiers : stockage local en MVP ou bucket objet type S3/MinIO en cible
	•	Authentification : JWT
	•	Déploiement : Docker
	•	CI/CD : GitHub Actions
	•	Hébergement : VPS, OVH, Azure ou équivalent

La base de données stocke uniquement l’URL ou le chemin du fichier.

8. Sécurité
Authentification
Authentification par email et mot de passe.
Le mot de passe doit être stocké sous forme hashée.
Autorisation
Contrôle des droits par rôle.
Données
Les actions sensibles doivent être historisées.
Fichiers
Les fichiers uploadés doivent être contrôlés :
	•	type autorisé
	•	taille maximale
	•	nommage sécurisé

9. Gestion des alertes
Une tâche planifiée vérifie régulièrement les chantiers.
Condition de retard :
Date actuelle > date fin prévue ET Statut différent de “Clôture”
Si la condition est vraie, une alerte active est créée ou maintenue.
