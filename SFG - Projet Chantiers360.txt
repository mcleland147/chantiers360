Spécifications Fonctionnelles Générales
Projet Chantiers360
Version 1.0 – MVP
1. Présentation
1.1 Contexte
BatiNova Travaux souhaite mettre en place une application web permettant de centraliser le suivi opérationnel des chantiers.
L’application devra remplacer les suivis actuellement réalisés à travers Excel, WhatsApp et les échanges par e-mail.

1.2 Objectifs
Permettre :
	•	le suivi des chantiers ;
	•	le suivi des équipes ;
	•	le suivi de l’avancement ;
	•	la gestion des réserves ;
	•	la gestion des photos terrain ;
	•	la détection des retards ;
	•	le pilotage de l’activité.

2. Profils utilisateurs
Direction
Droits :
	•	consultation ;
	•	reporting ;
	•	tableaux de bord.

Assistante administrative
Droits :
	•	création des chantiers ;
	•	modification des informations administratives.

Conducteur de travaux
Droits :
	•	gestion complète des chantiers ;
	•	affectation des équipes ;
	•	gestion des réserves ;
	•	validation des statuts ;
	•	clôture des chantiers.

Chef de chantier
Droits :
	•	consultation des chantiers ;
	•	mise à jour de l’avancement ;
	•	ajout de photos ;
	•	création de réserves.

3. Écran Dashboard Conducteur
Objectif
Permettre au conducteur de travaux de piloter l’ensemble de ses chantiers.
Composants
KPI
Afficher :
	•	Nombre de chantiers actifs
	•	Nombre de chantiers en retard
	•	Nombre de réserves ouvertes
Liste des chantiers récents
Colonnes :
	•	Référence
	•	Chantier
	•	Statut
	•	Date fin prévue
	•	Nombre de réserves
Alertes
Afficher :
	•	Retards
	•	Réserves critiques
Actions
	•	Consulter un chantier
	•	Filtrer
	•	Rechercher
Règles de gestion
RG-001
Un chantier est considéré en retard lorsque :
Date actuelle > Date de fin prévue
ET
Statut ≠ Clôturé

Objectif

	•	Ajouter une photo
	•	Créer une réserve

Contraintes UX

	•	Utilisable sur smartphone
	•	Temps cible inférieur à 30 secondes par action

14. Règles de gestion globales
RG-009
Le workflow chantier doit respecter l’ordre suivant :
Préparation → Planification → Démarrage → Réalisation → Réception → Clôture
RG-010
Un chantier ne peut pas être clôturé si une réserve est encore ouverte.
RG-011
Une date de réception doit être renseignée avant la clôture.
RG-012
Toute action importante doit être historisée.
RG-013
Toute modification doit être associée à un utilisateur authentifié.

15. Critères de réussite du MVP
Le MVP sera considéré comme validé lorsque les utilisateurs seront capables de :
	•	créer un chantier ;
	•	suivre son avancement ;
	•	gérer les réserves ;
	•	gérer les photos ;
	•	détecter les retards ;
	•	consulter les indicateurs de pilotage ;
sans recourir à des fichiers Excel externes.
